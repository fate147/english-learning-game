# 英语学习游戏 — 总计划（最终版）

> 面向小学三年级孩子的三科学习游戏。
> **英语版全部完成**（闯关 + 单词记忆 + 对话模式），准备进入语文+数学扩展阶段。
> 纯静态前端部署在 GitHub Pages，Supabase 做后端。

---

## 一、项目定位

给孩子用的学习游戏，家长管理学习进度、看统计、设奖励。不做考试，只做氛围。孩子在玩中学。

**当前阶段：** 英语三大模式已全部完成（闯关模式、单词记忆模式、对话模式），准备进入三科扩展阶段（语文+数学）。

## 二、技术栈

| 层 | 选型 | 备注 |
|------|------|------|
| 框架 | React 19 + Vite 8 | |
| 路由 | **HashRouter** | GitHub Pages 兼容，刷新不 404 |

## 九、核心数据流

### 9.1 分层架构

```
components/（.jsx，纯展示）
CSS 变量 --theme-h 驱动全局颜色
    ↕  通过 hooks 读写
hooks/（.js，副作用编排）
调 lib/ + 更新 context
    ↕
lib/ + engines/（.js，纯计算）
统一返回：{ data, error }
    ↕
Supabase DB / localStorage
```

**约束：**
- components/ 不直接 import supabase
- lib/ 和 engines/ 不 import React
- hooks/ 是唯一胶水层
- 所有 lib/* 统一 `{ data, error }` 格式，hooks 层处理 error 显示 Toast

### 9.2 登录流

```
Login → AuthProvider.login(email, pw) → signIn() → Supabase auth
  → setUser + localStorage 缓存 session → 跳转 /select-child
```

### 9.3 选孩子流

```
SelectChild → ChildProvider.fetchChildren()（5分钟缓存）
  → 展示 ChildCard → 点击 → selectChild(child)
  → 缓存 eng_last_child_id + eng_stars_{childId} → navigate('/game')
```

### 9.4 星星流

```
addStars(totalAdd, availAdd):
  1. 乐观更新本地（立即显示）
  2. 后台调用 Supabase RPC add_stars()
  3. 失败 → enqueue() 入离线队列
  4. 30s 轮询重传 → 成功 dequeue

spend(cost):
  1. 调用 RPC spend_stars()（SELECT FOR UPDATE 加锁）
  2. 成功 → 更新本地 available
  3. 失败 → 提示错误
```

### 9.5 离线流

```
首次打开: 有网→拉数据 / 无网→有缓存→正常玩 / 无缓存→OfflineGate 拦截
游戏过程: 只读写 localStorage
游戏结束: 尝试写 Supabase → 失败→enqueue
后台重传: startSync() → 每 30s → 有网逐条重传 → 成功移出
```

| 连续答对 3 题 | 额外 +1 |
| 连续答对 5 题 | 额外 +2 |
| 一局全对 | 额外 +3 |
| 每日第一次打开游戏 | +2 |
| 连续 7 天都玩 | +5 |
| 答错 | 不倒扣，断连击 |
| 对话模式答对一轮 | +1（无连击，无额外奖励） |

**两个星星字段分离：**
- `total_earned_stars`：总获得，只增不减，算等级
- `available_stars`：余额，兑换时扣减

**等级公式：** `LV = floor(totalEarnedStars / 10) + 1`

**颜色系统：** HSL 三段渐变（LV 1→15 绿→蓝，15→30 蓝→紫，30→50+ 紫→红）

## 十三、离线策略

```
首次打开：
  ├── 有网络 → 拉数据 → 写 localStorage → 正常游戏
  └── 无网络 → localStorage 空？
              ├── 是 → OfflineGate 拦截，显示"首次需要联网" + [重试连接] 按钮
              └── 否 → 正常游戏（本地有缓存）

游戏过程：
  只读写 localStorage，不直接查 Supabase

游戏结束：
  尝试写 Supabase
  ├── 成功 → 清本地临时记录
  └── 失败 → 进 localStorage 队列 + dirty 标记

后台重试：
  每 30 秒检测网络
  ├── 有网 → 逐条重传 → 成功移出队列
  └── 断网 → 静默等待
```

## 十四、开发原则

1. **先理解画面，再出方案** — 不自己猜需求
2. **旧版能复用的就不重造** — 词库、出题算法、计分规则直接搬
3. **目录按功能分层，不按科目** — `components/QuestionCard` 而非 `english/QuestionCard`
4. **所有文字标签抽成配置** — 不硬编码
5. **每完成一步提交一次** — commit message 标注改了哪些文件
6. **不改正在跑的生产功能** — 现有英语代码不动，三科扩展不破坏已有功能
7. **继承旧版好东西** — 角色动画效果（跳跃欢呼、摇头晃动）、反馈气泡视觉风格在新版保留

## 十五、已知遗留问题

> 以下问题在英语版中已存在，三科扩展前建议优先处理。

| # | 问题 | 影响 | 说明 |
|---|------|------|------|
| 1 | **三阶段进度未接入** | 高 | `wordProgressMap` 硬编码为 `{}`，复习/填词分级失效。出题算法无法按三阶段模型（熟悉→掌握→复习池）精准调度 |
| 2 | **DataPrefetch 只拉第一个孩子** | 中 | `children[0].child_id` 写死，非首个孩子进入时后台预取拉的是错误数据 |
| 3 | **WordMemory 星星不计入离线队列** | 中 | 单词记忆模式直接调 RPC 而非走 StarContext，断网时星星丢失 |
| 4 | **测试文件缺失** | 低 | `__tests__/` 为空，vitest 环境需改为 jsdom 才能跑 |
